Title: Reaching the Hardware Limit: Universal Decoupling and Zero-Cost Capability Routing in C++

Abstract:
Large-scale C++ systems suffer from the "Build Wall" — a state where shared headers
and centralized registries force a compilation tax that scales with team size. This
session introduces the Capability Routing Gateway (CRG), a linker-driven C++
framework that eliminates this tax by making modules completely anonymous to each
other: features self-register at link time, and the core discovers them only when
needed, with no shared headers and no Init() functions. The result is a fully
functional plugin system obtained for free as a side effect of the architecture.
The session is supported by an interactive WebAssembly simulator allowing
real-time stress-testing of up to 150,000 entities directly in the browser.

CRG is built on strictly orthogonal pillars — you pay only for the dimensions you
use. At its simplest, it is pure linker-driven discovery. Add a polymorphic
boundary, and you get interface-to-template binding. Add contextual axes (State,
Zone, Authority), and you get O(1) tensor-based routing using Horner's method. At
each level, the architecture stays C++-idiomatic and zero-allocation. When opaque
data transport is needed across module boundaries, an optional type-erased shell
breaks the fundamental C++ constraint that prevents virtual template methods.

In high-frequency execution paths, CRG mirrors the L1I/L1D split of modern CPUs
by caching resolved logic pointers in ECS components. Behavior transitions become
coordinate updates — the memory layout never changes, so the hardware prefetcher
never loses its stream. Benchmarks show this "Structural Immunity" effectively
doubles memory throughput at 10% mutation rates compared to traditional
archetype-based architectures. At low mutation rates (≤1%), ECS matches CRG —
the honest answer is that CRG's value scales with state volatility.

Attendees will learn how to:
* Achieve mutual anonymity: linker-driven NodeList registration with UniversalAnchor,
  supporting monolithic and DLL/hot-reload topologies with a single macro switch.
* Break the Virtual Deadlock using the optional ModelShell — a type-erased container
  inspired by Klaus Iglberger's Type Erasure pattern, stripped of all logic, with
  SBO guarantees and natvis debuggability.
* Project behaviors in O(1) using a flat CapabilityTensor indexed by DenseTypeID
  and a CapabilitySpace offset computed via Horner's method — no search, no branches,
  no virtual calls in the hot path.
* Integrate with any data-driven system using the Brain/Muscle pattern: ActiveCapability
  caches the resolved result at decision frequency; the hot loop invokes it at
  execution frequency with a direct function pointer.
* Identify the mathematical break-even point where CRG's structural immunity
  outperforms traditional ECS, using interactive simulation (up to 150k entities).

  How features self-register without shared headers. CapabilityBinding expands a

  Invoke/TryInvoke as the cold-path OOP illusion.

  framework: 0D (discovery) to ND (contextual routing). Collapsing sparse hashes

  into a flat CapabilityTensor.
* 35-45: ECS SYMBIOSIS — Immutable topology as a performance guarantee. Brain

* 54-55: CONCLUSION — Zero Coupling. Zero Search. Zero Migration. Stop fighting C++.
