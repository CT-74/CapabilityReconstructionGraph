Title: Beyond Archetypes: Decoupling State and Logic with Linker-Level Auto-Discovery

Abstract:
Large-scale C++ engines often suffer from the "Virtual-Template Deadlock": the friction between rigid inheritance hierarchies and the need for modular, high-performance logic. This leads to "Include Hell," massive recompilation times, and costly "Archetype Fragmentation" in ECS-based systems.

This session introduces the Capability Routing Gateway (CRG), an architectural framework that achieves total decoupling of state and logic through Stateless Behavior Projection. 

We move away from centralized registries and hardcoded dependencies. Instead, CRG leverages Static Anchor Points and C++17 auto-registration to allow gameplay plugins to "wire themselves" into the engine core across vcproj and DLL boundaries with zero manual registration. By treating logic as a transient view projected onto immutable data, we reach a level of modularity where adding or removing entire gameplay domains is a linker-level operation.

Attendees will learn how to:
* Eradicate Dependency Rot: Use static anchor points to create a plugin system where gameplay domains (Combat, AI, UI) remain completely invisible to each other and the engine core.
* Solve the Cross-DLL Frontier: Use the Stateless Shell (TES) as an "Ambassador" to transport data through generic interfaces across module boundaries without header pollution.
* Deep-Dive into the "Memory Wall": Analyze detailed benchmark data proving how decoupling leads to "Structural Immunity," saturating RAM bandwidth (30.8 Gi/s) on Apple M-Series hardware.
* Flatten Polymorphism: Map behaviors into an N-Dimensional Tensor, transforming complex contextual logic into a branch-predictor-friendly O(1) array access.

Outline:
* 00-05: The Dependency Wall: The cost of coupling; why "Include Hell" and Archetype Fragmentation are the same architectural flaw.
* 05-15: Anchor-Based Discovery: Implementing auto-registration via static anchor points in DLLs; making the linker do the "wiring" work across module boundaries.
* 15-25: Selective Projection: Dual-backend strategy. The Stateless Shell (TES) for flexibility vs. the Stateless DOD sniper for throughput.
* 25-40: The N-Dimensional Tensor & Logic Math: Transforming decisions into O(1) arithmetic; implementing "Space" evaluation via Horner’s method.
* 40-55: Data-Driven Performance Masterclass: Analyzing the "Archetype Poisoning" penalty. 60% throughput reclamation and the 30.8 Gi/s memory wall.
* 55-60: Q&A
