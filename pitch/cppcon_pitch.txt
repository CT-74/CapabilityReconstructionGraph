Title:  
A Runtime Capability Graph for External Behavior Composition in C++

Abstract:  
This talk introduces a structural approach to behavior composition in C++ where functionality is defined externally and discovered at runtime through a distributed capability graph.

Instead of relying on inheritance hierarchies, centralized registries, or ECS-style coupling, the system is built from independently defined components that self-assemble into a coherent resolution structure.

Each behavior is implemented as a definition that binds a model (identity) to an interface (contract). These definitions are not registered in a container—they are instantiated as independent objects that automatically link into the system. From these distributed nodes, a runtime capability graph emerges.

Behavior lookup is performed by combining a runtime identity and an interface, resolving through the graph without any centralized ownership or explicit mapping.

The talk progressively builds this model from a minimal self-registering structure to a full system where feeding two type lists—identities and behavior definitions—materializes a complete identity × behavior matrix.

The focus is not on specific C++ techniques, but on the architectural implications of replacing ownership-based design with composition through emergence.
